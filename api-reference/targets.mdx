## Overview

Targets define the configuration for stimulus generation (stim) and trajectory generation (rollout). They are composed from a base template plus per-target overrides.

**Related Endpoints**:
- [Stimulus generation](/api-reference/targets-stim)
- [Trajectory generation](/api-reference/targets-rollout)

## Endpoints

### List Targets

List all targets in a repository.

**Endpoint**: `GET /v1/repo/{repo_name}/targets`

**Request**:

```bash
curl -X GET "https://bapi.bread.com.ai/v1/repo/my_repo/targets" \
  -H "Authorization: Bearer $BREAD_API_KEY"
```

<ParamField path="repo_name" type="string" required>
  Repository name
</ParamField>

**Response**: `200 OK`

```json
{
  "targets": ["target_1", "target_2", "coding_target"]
}
```

### Get Target

Get target definition and metadata.

**Endpoint**: `GET /v1/repo/{repo_name}/targets/{target_name}`

**Request**:

```bash
curl -X GET "https://bapi.bread.com.ai/v1/repo/my_repo/targets/coding_target" \
  -H "Authorization: Bearer $BREAD_API_KEY"
```

<ParamField path="target_name" type="string" required>
  Target name
</ParamField>

<ParamField path="repo_name" type="string" required>
  Repository name
</ParamField>

**Response**: `200 OK`

```json

    "student_prompt": "user_prompt_python",
    "generators": [],
    "max_tokens": 150,
    "temperature": 1.0
  }
}
```

### Create or Update Target

Create or update a target. Targets are composed from a base template plus per-target overrides.

**Endpoint**: `POST /v1/repo/{repo_name}/targets`

**Request**:

```bash
curl -X POST "https://bapi.bread.com.ai/v1/repo/my_repo/targets" \
  -H "Authorization: Bearer $BREAD_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "target_name": "coding_target",
    "template": "default",
    "overrides": {
      "model_name": "Qwen/Qwen3-32B",
      "teacher_prompt": "system_prompt_coding",
      "student_prompt": "user_prompt_python",
      "generators": [
        {
          "type": "oneshot_qs",
          "model": "claude-sonnet-4-5-20250929",
          "numq": 10,
          "temperature": 1.0
        }
      ],
      "max_tokens": 150,
      "temperature": 1.0
    }
  }'
```

<ParamField path="repo_name" type="string" required>
  Repository name
</ParamField>

<ParamField path="target_name" type="string" required>
  Target name
</ParamField>

<ParamField path="template" type="string" required>
  Template name: `"default"` or name of existing target
</ParamField>

<ParamField path="overrides" type="object">
  Configuration overrides. See [Target Configuration](/sdk/configuration/target-config) for all fields.
</ParamField>

**Returns**: `TargetResponse`

---

### Batch Create or Update Targets

Create or update multiple targets in a single request.

**Endpoint**: `POST /v1/repo/{repo_name}/targets/batch`

**Request**:

```bash
curl -X POST "https://bapi.bread.com.ai/v1/repo/my_repo/targets/batch" \
  -H "Authorization: Bearer $BREAD_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "targets": [
      {
        "target_name": "target_alpha",
        "template": "default",
        "overrides": {"model_name": "Qwen/Qwen3-32B"}
      },
      {
        "target_name": "target_beta",
        "template": "target_alpha",
        "overrides": {"temperature": 0.7}
      }
    ]
  }'
```

## Configuration Fields

See [Target Configuration Reference](/sdk/configuration/target-config) for detailed documentation of all configuration fields.

### Key Fields

<ParamField path="generators" type="array">
  Data generation strategies. See [Generators](/sdk/configuration/generators).
</ParamField>

<ParamField path="model_name" type="string">
  Base model for rollout (required for rollout jobs)
</ParamField>

<ParamField path="teacher_prompt" type="string">
  Teacher prompt name - prompt that defines the target behavior being baked into the model.
  
  <Warning>
  **Deprecated**: The `u` field is deprecated in favor of `teacher_prompt`. Please use `teacher_prompt` instead.
  </Warning>
</ParamField>

<ParamField path="student_prompt" type="string">
  Student prompt name - simplified prompt given at inference. After baking, this triggers the teacher behavior. Can be empty (`""`) for always-on behavior.
  
  <Warning>
  **Deprecated**: The `v` field is deprecated in favor of `student_prompt`. Please use `student_prompt` instead.
  </Warning>
</ParamField>

<ParamField path="max_tokens" type="integer">
  Maximum tokens to generate
</ParamField>

<ParamField path="temperature" type="number">
  Generation temperature (0.0-2.0)
</ParamField>

<ParamField path="num_traj_per_stimulus" type="integer">
  Number of trajectories per stimulus
</ParamField>

<ParamField path="max_concurrency" type="integer">
  Maximum concurrent requests
</ParamField>

## Examples

### Simple Target with Hardcoded Questions

```bash
curl -X POST "https://bapi.bread.com.ai/v1/repo/my_repo/targets" \
  -H "Authorization: Bearer $BREAD_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "target_name": "simple_target",
    "template": "default",
    "overrides": {
      "generators": [
        {
          "type": "hardcoded",
          "numq": 3,
          "questions": [
            "Write a Python function to sort a list",
            "Create a REST API endpoint",
            "Implement a binary tree"
          ]
        }
      ],
      "model_name": "Qwen/Qwen3-32B",
      "teacher_prompt": "system_prompt",
      "student_prompt": "user_prompt"
    }
  }'
```

### Target with AI-Generated Questions

```bash
curl -X POST "https://bapi.bread.com.ai/v1/repo/my_repo/targets" \
  -H "Authorization: Bearer $BREAD_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "target_name": "ai_generated_target",
    "template": "default",
    "overrides": {
      "generators": [
        {
          "type": "oneshot_qs",
          "model": "claude-sonnet-4-5-20250929",
          "numq": 50,
          "temperature": 1.0
        }
      ],
      "model_name": "Qwen/Qwen3-32B",
      "max_tokens": 200,
      "temperature": 0.9,
      "teacher_prompt": "system_coding",
      "student_prompt": "user_task"
    }
  }'
```

<Tip>
The student prompt can be empty (`""`) for zero-token, always-on behavior. This proves the expertise is truly baked into the model weights!
</Tip>

    Generate trajectories (model responses) for the stimuli
  </Step>
  
  <Step title="Use in Bake">
    Reference the target's rollout data in a bake for model training
  </Step>
</Steps>
